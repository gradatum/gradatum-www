Memoria · cognitive memory backbone for AI agents

A place of

Gradatum is a self-hosted, embedded memory backbone for multi-agent AI systems. It cures memory rot in LLMs and lets Claude, Gemini, Codex and your home-grown agents share what they know — across sessions, across machines, across runs.

I · The problem

LLMs forget. Agents reinvent the wheel every session. Teams of agents can't share what they learn.

What's missing today
  • SaaS lock-in — your memory hosted on someone else's server
  • Heavy stacks — Postgres + pgvector + Neo4j just to remember things
  • Built for humans, not agents — no ACL, no multi-tenancy, no MCP
  • Transient context, not persistent KB — forgets between sessions
What Gradatum does

One Rust binary. SQLite + Markdown on disk. No PostgreSQL. No Redis. No SaaS. Pluggable LLM (or none). Multi-vault. Hierarchical ACL. Hybrid search — BM25 + ANN + PageRank. The Markdown files are the source of truth — not the index.

II · Properties

The Gradatum approach

I Embedded One Rust binary. No PostgreSQL. No Redis. No external services. apt install and you're running.
II Self-hosted Your memory, your machine. No telemetry. No vendor lock-in.
III LLM-agnostic Plug any OpenAI-compatible backend (Ollama, vLLM, llama.cpp, OpenRouter, Anthropic) — or run heuristic-only with no LLM at all.
IV Multi-vault Separate main from staging and bench-* vaults for testing, migration, A/B prompts. Atomic swap when ready.
V Hierarchical ACL Bearer-scoped access to memory loci. Configure from presets (flat, hierarchical, multi-project, team) or write your own.
VI Multi-storage OpenDAL abstraction — 8 backends: Local FS, S3/R2, Azure, GCS, WebDAV, SFTP, HDFS, IPFS. NFS explicitly rejected.
VII Markdown truth Notes are Markdown files with YAML frontmatter. Readable by humans and by cat. The database is an index, not the source of truth.
VIII Hybrid search BM25 (Tantivy) + semantic ANN (sqlite-vec) + PageRank graph + cross-encoder rerank. Multi-signal fusion via RRF.
III · Architecture

One binary. Four planes. Markdown on disk.

A stateless façade speaks HTTP and MCP to your agents. A worker drains a SQLite-backed queue. One vault per instance is the default — staging and bench-* vaults are first-class for migration and A/B testing. 

        AI agents · coding assistants · orchestrators
              ↓  MCP / HTTP / CLI  (RFC-0003: :19090)
        ┌─────────────────────────────────────────┐

        │  /api/v1  /mcp  /sse  /health  /admin  │

IV · Vocabulary

Six levels, named with care.

Borrowed from Cicero's ars memoriae: agents place their memories in loci, mental locations of an imagined palace. Agents don't share rooms — they share places of memory.

Vault
The technical backing store (SQLite + Tantivy + Markdown). Multi-vault first-class — main + staging + bench-*.
Locus
A logical subdivision of a vault, isolated by ACL. From Cicero's ars memoriae — the mental location where an image is placed.
Section
One of 10 cognitive categories: decisions, architecture, debug, reasoning, feedback, lessons-learned, retrospectives, experiments, agent-issues, reference.
Note
Atomic Markdown file with YAML frontmatter. ULID identity. SHA-256 content hash for drift detection.
Bearer
An authenticated identity with read/write ACL patterns over loci. Configured via presets or custom bearer.toml.
Preset
A template configuration shipped in examples/presets/ — flat, hierarchical, multi-project, team.
V · Multi-storage

Eight backends. One abstraction.

Gradatum uses Apache OpenDAL as a unified storage abstraction layer. Switch backends without touching application code. NFS is explicitly rejected — see RFC-0002 §8.

  • Local FS primary
    fs://

    Default. NVMe local only — NFS explicitly rejected per RFC-0002 §8.

  • S3 / R2 primary
    s3://

    AWS S3, Cloudflare R2, MinIO, Backblaze B2.

  • Azure Blob secondary
    azblob://

    Azure Blob Storage via OpenDAL azblob service.

  • GCS secondary
    gcs://

    Google Cloud Storage. Service account or ADC auth.

  • WebDAV secondary
    webdav://

    Self-hosted WebDAV (Nextcloud, Nginx, Apache).

  • SFTP secondary
    sftp://

    SSH File Transfer Protocol. Useful for NAS with SSH.

  • HDFS experimental
    hdfs://

    Hadoop Distributed FS. Enterprise / data-lake.

  • IPFS experimental
    ipfs://

    InterPlanetary File System. Decentralised, content-addressed.

Example — gradatum.toml
# Per-vault storage configuration
[vaults.main.storage]
backend = "s3"
bucket  = "my-gradatum-vault"
region  = "us-east-1"
root    = "/gradatum/main"

[vaults.staging.storage]
backend = "fs"
root    = "~/.gradatum/vaults/staging"  # local NVMe for testing

# NFS explicitly rejected — see RFC-0002 §8
# backend = "nfs"  ← will fail at startup with nfs_check.rs guard

VII · RFCs

Design decisions, written down.

All major architectural choices are captured as RFCs. Three accepted so far.

  • RFC-0001 accepted
    Versioning gradatum-core

    Trait stability tiers and versioning policy for gradatum-core. Defines stable/unstable/internal tiers and SemVer rules per tier.

    Read →
  • RFC-0002 accepted
    Cross-platform support

    B-light tiered model: Linux x86_64 primary (full CI + runtime), Windows x86_64 secondary (cross-compile mingw-w64, manual pre-release). 13 portability rules R1–R13.

    Read →
  • RFC-0003 accepted
    HTTP API surface and MCP integration

    Single-port :19090 path-prefix routing (/api/v1 /mcp /sse /health /admin). gradatum-mcp-stub as stdio bridge for Claude Code, Claude Desktop, ZeroClaw.

VIII · Quick start

Build from source today. apt install with Phase 4.

Clone & build
# clone & build
git clone https://github.com/gradatum/gradatum
cd gradatum && cargo build --release --workspace

# initialize a vault with the hierarchical preset
gradatum-admin init --preset=hierarchical \
  --projects=projecta,projectb

# start the server (Phase 2.0+)
systemctl --user start gradatum-server
Write & search
# write a note
gradatum write --locus=projecta/backend \
  --section=decisions \
  "Use ULID for stable note identity"

# search across loci
gradatum search "ULID identity" --locus="projecta/*"

# list vaults
gradatum-admin vault list
MCP integration (Phase 2.0)
# gradatum.toml — MCP stub for Claude Code
[[mcpServers]]
name    = "gradatum"
command = "gradatum-mcp-stub"
args    = ["--server", "http://localhost:19090"]
env     = { GRADATUM_BEARER = "your-bearer-token" }

# Or via HTTP directly (API)
curl http://localhost:19090/health
curl http://localhost:19090/api/v1/notes/search \
  -d '{"query":"ULID","locus":"projecta/*"}'

One-liner installer (curl -fsSL https://gradatum.org/install.sh | bash) and .deb packages are planned for Phase 4.
